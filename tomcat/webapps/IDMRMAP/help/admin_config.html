Administrator Configuration

Use the following information to configure access to the Role Mapping Administrator. If you need additional help, see the Identity Manager Role Mapping Administrator 4.0 Installation and Configuration Guide.

  • Vault Display Name: Specify a display name for the Identity Vault in the Role Mapping Administrator.

  • Role Vault Address: Specify the DNS address of the Identity Vault. IP addresses are accepted but not recommended.

  • Role Vault Port: Specify the Identity Vault port. The default port is 389 or 636 for SSL.

  • Use SSL: Select this option to connect to the Identity Vault through SSL.

  • Admin DN: Specify the distinguished name (DN) of an Identity Vault administrator user. The administrator user provides a proxy through which the Role Mapping Administrator can perform LDAP operations in the Identity Vault.

  • Admin Password: Specify the password for the Identity Vault administrator.

  • Root User Container: Browse to the root container for the user objects in the Identity Vault.

    The container must be specified by the fully qualified LDAP DN. For example:

    ou=users,ou=data,o=novell

  • User App Driver DN: Browse to the User Application driver located in the Identity Vault.

    The driver must be specified by the fully qualified LDAP DN. For example:

    cn=UserApp1,ou=IDMDrivers,o=novell

  • Driver Discovery DN: Browse to the root location to search for drivers. For example, if you only have one driver set, specify it in this field. If you have multiple driver sets, specify the container that holds the driver sets.

    If this field is left blank, an LDAP search is performed of the entire Identity Vault.

  • Access Manager Logout URL: Specify the URL for the Access Manager Identity Server. This is an optional field to allow for allow for simultaneous logout from the Role Mapping Administrator and Access Manager.

  • Resource Category: Specify the default category with which the resources will be created.
